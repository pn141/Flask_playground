Hello { username }, now that you have authenticated, you can make a number of changes to your account.

However, the main idea behind Flask-playground is for it to be changed.

Changing this project would essentially require to:

  • Make changes to the database

    • Flask-playground only has a model for users. You might want to add a "note" table to create a todo list for example. This would require that you add a note_model.py file to the app\models folder.

  • Create a form

    • I would recommend to create a "notes" folder inside the "app" then to add a "form.py" file to it to reprent your web form.

  • Create a template for HTML rendering of the form

    • a "templates" folder already exists under "app"; you should create a "notes" folder to keep your html templates separate from the rest of the application.

  • Create a view

    • I would recommend to create a "views.py" inside the "app/notes" folder previously created.

Extending the folder structure as proposed will help you add, debug, test or discard your changes easily. Notice how adding functionality to create notes would not require to edit or change any of the existing part of the project.
